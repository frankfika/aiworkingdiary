为什么是 Web3？

Web3 让你在没有平台背书的情况下，证明数字资产属于你。

  • 无需客服、无需身份证明，只要私钥签名，链上即承认权利。
  • 规则写进协议，数据公开可查，无人可单方面删改。
  • 从“平台说了算”到“我的数字权利，由我来定义”。
案例：Web2 游戏公司可削弱你的装备；Web3 NFT 属性上链，未经你签名不可更改。
这一页我们用一个游戏装备的例子来说明 Web3 的价值：在 Web2 里，游戏公司可以削弱你的装备；而在 Web3 里，NFT 的属性写在链上，未经你签名就无法更改，这就是“数字所有权”的核心。

数字所有权与状态改变

区块链是一台记录“状态”的机器：谁拥有什么、授权给谁、NFT 归谁。

  • 每次交易 = 请求把状态从 A 变到 B，并用私钥签名同意。
  • 状态改变不可逆，无客服可回滚。

签名前自问三句

这会改变什么状态？谁会受影响？改变后能否撤销？

这一页我们强调“状态”概念：区块链本质上是一台状态机，谁拥有什么、授权给谁、NFT 归谁，都是状态。每次交易就是请求把状态从 A 变到 B，并且必须你用私钥签名同意。记住，状态改变不可逆，签名前务必自问三句：这会改变什么状态？谁会受影响？改变后能否撤销？

公钥 / 私钥 / 地址

私钥

256 位随机数 → 万能钥匙
绝对保密，泄露即失去所有权。

公钥 / 地址

可公开，用于验证签名与收款
无法反推私钥。

常见泄露：助记词拍照同步云端、复制粘贴给“客服”、木马记录剪贴板。

守住私钥 = 守住“同意的权利”。

账户类型：EOA vs 合约账户

  • EOA：私钥控制，可主动发交易，无代码。
  • 合约账户：代码控制，无私钥，需被触发。
警惕：税率可调、可升级代理、无限 owner 权限 → 项目方可随时改规则卷款。

交互前必查

有无 owner？权限多大？是否可升级？有无时间锁/多签？

交易结构：from / to / value / data

value=0 不代表安全 —— 资产转移常编码在 data 字段。

  • ERC-20 转账：data = transfer(地址, 金额)
  • 授权：data = approve(合约, 额度)
只看 value 会被误导；务必读懂 data 里调用的函数与参数。

状态改变 vs 事件 Logs

  • 状态：链上真相，余额、授权、NFT 所有者。
  • 事件 Logs：操作明细，方便前端检索，非状态本身。

例：Transfer 事件方便浏览器展示流水，但余额变化已写入状态映射。

安全视角

只看事件不够，必须核对状态是否如预期改变。

区块与共识：确认 & 最终性

  • 交易被纳入区块 ≠ 最终，可能重组。
  • 确认数↑ → 重组成本↑，通常 12 块以上视为高概率不可逆。
案例：CryptoKitties 高峰期，低 Gas 交易等待数十区块才确认。

大额转账 / 关键操作：等待足够确认数再执行下一步。

Gas 费用结构

伦敦升级后：GasUsed × (BaseFee + PriorityFee)

  • BaseFee 自动调节，拥堵↑则↑，燃烧销毁。
  • PriorityFee（小费）吸引矿工/验证者优先打包。
NFT 铸造 Gas 战：BaseFee 瞬间飙升 10 倍，未及时调整导致交易卡住。

节省技巧

非紧急交易选低峰时段；钱包支持“最高费用”保护避免超额付费。

nonce 与失败定位

  • nonce 顺序：必须严格递增，跳号会卡住后续交易。
  • 常见失败：Out-of-Gas（GasLimit 不足）、Revert（合约回滚）。
案例：复杂 DeFi 交互少设 GasLimit → 交易失败仍烧掉上百美元 BaseFee。

排查步骤

看状态→查 nonce→对比 GasLimit 与实际用量→读 revert 消息。

L1 / L2 与链 ID

  • 链 ID：防止重放攻击的“网络身份证”，主网 1、Sepolia 11155111。
  • L2：Rollup 把交易批量写入 L1，降低费用、提高速度。
2016 年 ETC 分叉后无链 ID 保护 → 交易在主网/ETC 双向重放，用户资产受损。

签名前确认链 ID，避免测试网交易被重放到主网。

多链与网络切换

同一地址可在多条链存在，但资产、授权、nonce 互不共享。

  • 切换网络 ≠ 资产自动迁移，需跨链桥或手动转账。
  • RPC 节点故障 → 余额加载不出，换节点即可。

演示计划

MetaMask 手动添加 Sepolia → 领水 → 查看浏览器状态差异。

托管 vs 自托管

交易所托管

  • 私钥由平台保管
  • 可找回密码
  • 合规、客服支持
  • 单点故障、被冻结风险

自托管

  • 私钥完全自持
  • 无客服、无冻结
  • 自由跨链/DeFi
  • 泄露即永久损失

大额长期 → 自托管 + 硬件钱包；小额交易 → 托管便利。

安全备份与最小授权

案例：2021 年 Infinite Approval 漏洞，用户无限授权 DEX 被黑客一次性转空。
  • 离线手写助记词，防火金属板，多地分散。
  • 授权额度 ≤ 所需，用“自定义额度”替代“无限”。
  • 硬件钱包：私钥永不触网，即使电脑中毒也无法读取。

备份清单

手写 → 密封袋 → 两处异地 → 测试恢复 → 再存大额。

常见钓鱼与假网站识别

  • 域名拼写陷阱：metamask.io → metarnask.com
  • 谷歌广告置顶：钓鱼站买品牌词，排官网之上。
  • 推特空投链接：伪造“官方支持”引导输入助记词。
2022 年虚假 OpenSea 邮件 → 用户连接钱包 → 授权“transferFrom”被盗 100+ ETH。

三秒自检

域名拼写 → HTTPS 证书 → 是否要求助记词；书签直达，勿点广告。

演示：添加 Sepolia 测试网 & 签消息

  1. Chainlist 一键添加 Sepolia，链 ID 11155111
  2. 领水 faucet 获取测试 ETH
  3. 对比 personal_sign vs EIP-712 结构化签名
盲签 hex 数据 → 可能签名恶意交易；EIP-712 可读字段降低风险。

现场练习：扫码领水 → 签名 → 浏览器验证。

安全备份

正确做法

  • 纸笔手写，离线记录
  • 制作 2-3 份备份
  • 分散存放不同位置
  • 防火防水密封保存
  • 测试恢复确认正确

危险操作

  • 拍照存相册
  • 保存在备忘录或云端
  • 通过聊天工具发送
  • 只记一份
  • 从未测试恢复
最小授权原则
  • 拒绝无限授权，只给必要额度
  • 操作完成后撤销授权
  • 定期审查授权列表（revoke.cash）
  • 检查合约是否可信
