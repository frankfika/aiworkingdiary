# 第1课 Web3/区块链认知 + 钱包与安全｜PPT逐页大纲充实版（约92分钟，18页）

总览：本课统一采用测试网演示（Sepolia），不展示真实私钥与资产。不做投资建议，仅做技术与安全教育。每页包含：标题、时长、学习目标、关键点、视觉建议、讲解提示、互动/演示、风险提醒、对应逐字稿位置。

P1 标题与免责声明｜3分钟
- 学习目标：明确课程边界与学习产出；建立“谁签名谁负责”的底线。
- 关键点：课程范围（技术与安全）、测试网演示、时间结构总览。
- 视觉建议：标题页 + 小字免责声明；时间轴总览（18页/78分钟）。
- 讲解提示：强调“不含投资建议”“不展示私钥”；点出主线“状态改变”。
- 互动：提问“你觉得链上‘签名’意味着什么？”收集两位同学回答。
- 风险提醒：课堂不引导真实资金操作，不分享私钥片段。
- 对应逐字稿：P1。

P2 为什么是 Web3｜4分钟
- 学习目标：理解 Web3 的价值命题：可验证的数字所有权与开放接口。
- 关键点：Web2 与 Web3 的控制差异；公共账簿 vs 私账本；全球结算属性。
- 视觉建议：对比图（Web2 平台控制 vs Web3 协议共识）；“开放接口生态”图示。
- 讲解提示：用生活类比（银行私账本 vs 公开账簿）；强调“无需平台背书的所有权证明”。
- 互动：让同学举一个“平台说了算”的例子（封禁账号、冻结资金）。
- 风险提醒：避免把“开放”误读为“无风险”；开放意味着自我负责。
- 对应逐字稿：P2。

P3 数字所有权与状态改变｜4分钟
- 学习目标：掌握“状态（state）”这一核心概念；知道交易签名改变状态。
- 关键点：状态示例（余额、合约变量、所有权）；签名是“同意改变”。
- 视觉建议：状态机图（A→B）；授权额度从 0→100 的示例。
- 讲解提示：强调“理解状态改变比记术语更重要”。
- 互动：请同学说一个他们能想到的状态改变例子；板书 2 个。
- 风险提醒：任何状态改变都需要签名；谨慎每一次签名弹窗。
- 对应逐字稿：P3。

P4 公钥/私钥/地址关系｜7分钟
- 学习目标：理解钥匙-公钥-地址的不可逆链条；明确私钥保密的重要性。
- 关键点：单向函数不可逆；签名用私钥、验证用公钥；地址派生。
- 视觉建议：流程图（私钥→公钥→地址）；攻击面示意（拍照/云盘/聊天工具）。
- 讲解提示：列举错误备份方式与正确备份清单（纸笔离线、分散、硬件钱包）。
- 互动：让同学指出“拍照助记词”为何危险；给出两条理由。
- 风险提醒：备份时关闭摄像头/屏幕录制；环境干净再操作。
- 对应逐字稿：P4。

P5 账户类型：EOA vs 合约账户｜5分钟
- 学习目标：区分谁决定行为（你 vs 代码与权限）。
- 关键点：EOA 与合约账户差异；Owner 权限、增税/黑名单、Proxy 升级风险。
- 视觉建议：权限图谱（Owner、Proxy、Implementation、可修改参数）。
- 讲解提示：建立“看到权限就问谁能改”的习惯；举 1-2 个常见危险参数。
- 互动：问“如果合约可升级，风险会是什么？”引导出实现合约可变更。
- 风险提醒：未 `renounce` 的 Owner 槽位、升级代理可更改逻辑。
- 对应逐字稿：P5。

P6 交易结构（from/to/value/data）｜5分钟
- 学习目标：看懂交易核心字段，明确“ERC-20 真正金额在 data 中”。
- 关键点：from/to/value/data；Wei → ETH 单位；方法选择器与参数编码。
- 视觉建议：区块浏览器截图标注；`transfer(address,uint256)` 编码示例。
- 讲解提示：解释“value=0 也可能在转账”；定位状态改变位置。
- 互动：展示截图，问“这是合约调用还是普通转账？”引导识别。
- 风险提醒：不要只看 value 忽略 data；避免误判资产变动。
- 对应逐字稿：P6。

P7 状态改变 vs 事件 Logs｜4分钟
- 学习目标：彻底区分“写入状态”与“记录事件”。
- 关键点：状态是核心数据；事件是索引日志；Transfer 事件与余额变化关系。
- 视觉建议：并列图（状态存储 vs 事件日志）；交易详情对比。
- 讲解提示：强调“事件不是状态”；前端常依赖事件但不能替代状态。
- 互动：让同学说出事件的用途（前端/分析工具）；板书 2 条。
- 风险提醒：只看事件可能误判；需要结合状态与执行结果。
- 对应逐字稿：P7。

P8 区块与共识（确认/最终性）｜7分钟
- 学习目标：理解确认与最终性差异；知道为何等待多确认。
- 关键点：PoW 概率最终性 vs PoS 经济最终性；确认阈值的场景化权衡。
- 视觉建议：区块链接图；不同确认策略示意（交易所 vs 协议）。
- 讲解提示：“越多确认越不易回滚”，但牺牲时间；给出真实策略对比。
- 互动：问“如果你是交易所，设置几次确认？”引导出金额敏感度。
- 风险提醒：低确认收款可能被回滚；大额务必提高确认要求。
- 对应逐字稿：P8。

P9 Gas/费用结构（基础费/优先费）｜6分钟
- 学习目标：掌握 EIP-1559 费用构成与加速逻辑。
- 关键点：Base Fee 动态调整；Priority/Tip 加速；拥堵成本与等待策略。
- 视觉建议：费用条形图；拥堵曲线与费率波动示意。
- 讲解提示：在高拥堵不做批量操作；讲“等一等更划算”的场景。
- 互动：问“你会在什么情况下提高优先费？”总结 2 条。
- 风险提醒：盲目加 Tip 可能烧钱；优先费与总价联动要审慎。
- 对应逐字稿：P9。

P10 nonce 与失败定位（Out-of-Gas/Revert）｜6分钟
- 学习目标：知道挂起交易影响后续；会读失败类型并排错。
- 关键点：nonce 顺序；Out-of-Gas vs Revert；替换/取消交易技巧。
- 视觉建议：失败示例截图；替换交易（同 nonce 更高费用）流程图。
- 讲解提示：给出排错清单：状态→事件→错误信息→替换/取消。
- 互动：问“遇到 Revert 你第一步看哪里？”引导看错误信息。
- 风险提醒：错误替换可能导致重复支付；仔细核对 nonce 与费用。
- 对应逐字稿：P10。

P11 L1/L2 与链 ID｜4分钟
- 学习目标：理解结算层与扩容层；链 ID 防重放的作用。
- 关键点：L1 保障最终性；L2 扩容批量提交；链 ID 区分签名域。
- 视觉建议：分层架构图（L1/L2）；链 ID 表（示例：1/11155111）。
- 讲解提示：轻触乐观与零知识 Rollup 差异，不深入技术细节。
- 互动：问“为什么不同链 ID 很重要？”引导重放风险。
- 风险提醒：错误网络签名可能落入钓鱼 RPC；谨慎自定义网络。
- 对应逐字稿：P11。

P12 多链与网络切换（Sepolia 演示计划）｜4分钟
- 学习目标：掌握安全添加与切换测试网的流程。
- 关键点：使用 Chainlist 添加；核对 RPC 与链 ID；切换提示识别。
- 视觉建议：Chainlist 搜索与添加截图；钱包网络切换提示。
- 讲解提示：说明测试网的练习价值与风险边界（不涉及真实资产）。
- 互动/演示：现场展示添加与切换；留意提示信息。
- 风险提醒：警惕假 Chainlist/假 RPC；始终走官方入口。
- 对应逐字稿：P12。

P13 托管 vs 自托管（权责对比）｜5分钟
- 学习目标：形成场景化选择与组合配置思路。
- 关键点：托管的便捷与平台风险；自托管的掌控与自我责任。
- 视觉建议：权责对比表（找回能力、操作门槛、风控责任）。
- 讲解提示：新手路径：先合规托管→逐步自托管→硬件钱包大额冷存。
- 互动：问“你目前更倾向哪种？为什么？”收集 2-3 个理由。
- 风险提醒：集中单点风险；组合配置分散风险更稳健。
- 对应逐字稿：P13。

P14 安全备份与最小授权原则｜8分钟
- 学习目标：掌握备份清单与授权管理；建立定期撤销习惯。
- 关键点：离线记录、分散备份、硬件钱包；只给必要额度、谨慎无限授权。
- 视觉建议：备份流程图；授权撤销界面示例（不现场执行，仅说明）。
- 讲解提示：讲 1 个去敏失败案例（无限授权 + 钓鱼站），给出改进清单。
- 互动：问“你会如何设置授权额度？多久检查一次？”
- 风险提醒：无限授权是高危；只走官方入口做授权与撤销。
- 对应逐字稿：P14。

P15 常见钓鱼与假网站识别｜4分钟
- 学习目标：建立快速识假小清单；会做二次验证。
- 关键点：URL/证书/来源；零费水龙头诱导；插件过权。
- 视觉建议：真假页面对比（拼写/域名后缀/权限提示）。
- 讲解提示：强调不从社交直达钱包连接页；多来源交叉验证。
- 互动：对比两张图找出可疑项；请 1 位同学说明理由。
- 风险提醒：假链接与假客服常用私聊；任何恢复服务都要怀疑。
- 对应逐字稿：P15。

P16 轻量演示：添加 Sepolia 与签消息｜8分钟
- 学习目标：体验签名弹窗与网络切换；理解“签名≠必然转账”。
- 关键点：添加网络→切换→签一条消息；解读提示字段与权限。
- 视觉建议：现场演示屏幕；事先准备截图防网络不稳。
- 讲解提示：不展示私钥；不进行真实资金转移；仅测试网。
- 互动/演示：签一条消息后请同学描述看到的字段。
- 风险提醒：任何签名都应明确允许的操作；拒绝看不懂的弹窗。
- 对应逐字稿：P16。

P17 互动问答｜4分钟
- 学习目标：用复述巩固；纠正常见误区（事件=状态）。
- 关键点：私钥不能共享的理由；失败定位第一步；托管 vs 自托管权衡。
- 视觉建议：板书关键词（状态/签名/nonce/Gas/最终性/链 ID）。
- 讲解提示：快速轮答，控制每题 30-45 秒；把答案归纳为 2-3 要点。
- 互动：现场问答；请 3 位同学回答不同问题。
- 风险提醒：自信心不等于安全；按清单执行更可靠。
- 对应逐字稿：P17。

P18 课后作业与资料（含收尾与预告）｜4分钟
- 学习目标：将知识落到可执行事项；为下一课铺垫。
- 关键点：备份策略与地址说明（不含敏感信息）；Sepolia 小额转账练习；记录 hash 并观察事件/状态。
- 视觉建议：作业卡片；官方链接列表（MetaMask/Chainlist/Etherscan）。
- 讲解提示：再次强调测试网练习与安全边界；预告下一课内容（读交易、看状态与事件、定位费用与失败）。
- 互动：请 1 位同学复述作业步骤；提醒下次课会抽查心得。
- 风险提醒：切勿把助记词写进作业；仅提交观察与思考。
- 对应逐字稿：P18。

讲师素材与准备清单
- 素材：区块浏览器截图（普通转账/合约调用/失败示例）、Chainlist 添加与钱包切换截图、授权页面说明截图（不操作）、真假页面对比图。
- 设备：稳定网络；演示账户（测试网）；硬件钱包展示机身（不连接、不演示真实签名）。

- 演示统一：一个钱包（MetaMask）+ 一个测试网（Sepolia）；仅测试网演示。

术语中英对照（讲师手册）
- 状态（State）；签名（Signature）；私钥（Private Key）；公钥（Public Key）；地址（Address）
- 账户（EOA/合约账户）；交易（Transaction）；交易数据（Calldata）；Nonce（交易序号）
- Gas；基础费（Base Fee）；优先费/小费（Priority Fee/Tip）；最大费用上限（MaxFeePerGas）
- 区块（Block）；最终性（Finality）；Layer 1 / Layer 2（L1/L2）；链 ID（Chain ID）
- 事件（Event）；主题（Topic）；EIP-1559；EIP-155；EIP-2718；EIP-2930；EIP-712
- ERC-20；ERC-721；水龙头（Faucet）；秘密恢复短语（SRP）
